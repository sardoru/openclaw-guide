Step 1.8 — Auto-Stub Script

Ensure daily logs never have gaps

tools/ensure-daily-log.sh
#!/bin/bash
DATE="${1:-$(date +%Y-%m-%d)}"
WORKSPACE="/path/to/your/workspace"  # Change this
LOG_FILE="$WORKSPACE/memory/${DATE}.md"

if [ -f "$LOG_FILE" ]; then
    echo "Daily log already exists: $LOG_FILE"
    exit 0
fi

echo "No daily log for $DATE  creating stub..."
{
    echo "# ${DATE} (auto-generated stub)"
    echo ""
    echo "_No main-session activity recorded._"
    echo ""
    echo "## Notes"
    echo "- (no activity recorded)"
} > "$LOG_FILE"
echo "Created stub: $LOG_FILE"

#!/usr/bin/env bash
# brain-ingest  Extract structured knowledge from video/audio/articles
# Usage:
#   brain-ingest "https://youtube.com/watch?v=..."
#   brain-ingest --vault silverleafe --category research "URL"

set -euo pipefail

WORKSPACE="/path/to/workspace"  # Change
VECTOR_DIR="$WORKSPACE/tools/vector-memory"

declare -A VAULTS=(
  ["main"]="/path/to/your/obsidian/vault"  # Change
)

VAULT_KEY="main"
CATEGORY="research"
INPUT=""

while [[ $# -gt 0 ]]; do
  case "$1" in
    --vault) VAULT_KEY="$2"; shift 2 ;;
    --category) CATEGORY="$2"; shift 2 ;;
    *) INPUT="$1"; shift ;;
  esac
done

VAULT_PATH="${VAULTS[$VAULT_KEY]}"

# Step 1: Download/transcribe (yt-dlp + whisper for video)
# Step 2: Extract claims via LLM
# Step 3: Write to Obsidian vault
# Step 4: Write to memory/ingested/
# Step 5: Reindex vector memory

echo "brain-ingest: Processing $INPUT -> $VAULT_PATH/$CATEGORY/"

Evaluator Scoring Rubric

4 weighted criteria with hard minimums

CriterionWeightMin to PassWhat It Checks
Design Quality30%5/10Cohesive palette, typography, spacing, distinct mood
Originality25%5/10Custom decisions vs AI slop patterns
Craft20%6/10Font scale, spacing base, WCAG contrast, hover states
Functionality25%7/10Every button works, forms submit, data persists

Overall pass: ALL criteria meet minimum AND weighted average >= 6.0

Anti-Leniency Rules

Baked into the evaluator prompt

  1. Never use "minor issue" to dismiss a finding. If you found it, score it.
  2. Never approve what you haven't tested. If you can't click it, it doesn't count.
  3. Stubbed features = FAIL. A button that logs to console is not implemented.
  4. "It looks like it works" is not evidence. Navigate, click, verify.
  5. Don't grade on effort. Time spent doesn't affect the score.

Importance Scoring

The 5-signal write gate

SignalMax PtsDescription
Novelty2First time encountering this?
Impact3How much did it affect outcomes?
Reusability2Will this apply to future tasks?
Difficulty2How hard to figure out?
Surprise1Was the result unexpected?
Threshold: 5+ = log the episode. Below 5 = skip. Score 8+ = highlight in daily note.
Decay Formula

Recent + important + relevant memories surface first

scoring
final_score = (relevance x 0.4) + (importance x 0.3) + (recency x 0.3)

recency = e^(-0.693 x age_days / half_life_days)

# With 30-day half-life:
# Today:    1.0
# 1 week:   0.85
# 30 days:  0.5
# 90 days:  0.125
# 180 days: 0.016

Generalizability Scoring

"Would this approach matter if this task disappeared?"

Before logging an episode, apply the generalizability test: if the specific task never came up again, would the lesson still be valuable? This filters task-specific noise from transferable knowledge.

Add to AGENTS.md
### Generalizability Test

Before logging a lesson, ask: "Would this approach matter if this specific task disappeared?"

- YES -> Log it. This is transferable knowledge.
- NO  -> Skip it. This is task-specific detail that won't compound.

Examples:
- "nginx 'failed to bind' actually means config parse error" -> YES (applies to all nginx upgrades)
- "The button was 2px off on the dashboard" -> NO (one-time fix, no lesson)

Security Hardening — 6-Layer Defense
Every untrusted input (X articles, web pages, email, chat) is a prompt injection surface. Six layers of defense, cheapest first, ~1,100 lines of bash.
securityprompt-injectionsanitizerredactiongovernance
Inspired by Matthew Berman's "Teaching OpenClaw to not get Hacked" (1,542 bookmarks). No single check catches everything. Layers run cheapest first — most attacks die at Layer 1 (free, instant regex) and never reach Layer 2 (an LLM call).
Architecture

6 layers, cheapest first

Untrusted input → Layer 1: Text sanitization (regex, Unicode cleanup) [free, instant] → Layer 2: Frontier scanner (LLM risk scoring) [$0.001/call] → Layer 3: Outbound content gate (leak detection) [free, instant] → Layer 4: Redaction pipeline (PII, secrets) [free, instant] → Layer 5: Runtime governance (spend caps, loop detection) [free, instant] → Layer 6: Access control (path guards, URL safety) [free, instant]
Layer 1 — Text Sanitizer

Strips invisible attacks before any LLM sees the text

Attack TypeWhat It DoesDefense
Invisible UnicodeZero-width chars hide instructions between visible textStrip all zero-width, RTL/LTR marks, soft hyphens, BOMs
HTML entities&#115;ystem decodes to system, bypasses regexDecode entities before pattern matching
Cyrillic lookalikesаеорс look identical to Latin aeopcNormalize to Latin equivalents
Role markers"ignore previous", "system:", "admin mode"Pattern match 20+ injection phrases, score each hit
Base64 blocksEncoded hidden instructionsDetect long base64 strings
bash
# Sanitize untrusted text:
echo "untrusted content" | bash tools/security/sanitize.sh
# Exit 0 = clean, Exit 1 = blocked
# Stats to stderr: {"role_markers": 5, "total_score": 10, "blocked": true}
Layer 2 — Frontier Scanner

LLM-based semantic injection detection

Uses a dedicated classifier model (not your agent). Returns structured risk score 0-100. Overrides model verdict if score contradicts it. Fail behavior configurable per source type.

bash
# email = fail closed (block on API error)
echo "email body" | bash tools/security/frontier-scan.sh --source email

# web = fail open (allow on API error)
echo "web content" | bash tools/security/frontier-scan.sh --source web
Layer 3 — Outbound Gate

Catches leaked secrets before they leave the system

Scans every outbound message (Telegram, WhatsApp, email) for API keys, internal paths, private IPs, exfiltration attempts via markdown images, and injection artifacts.

Layer 4 — Redaction Pipeline

Auto-strips PII and secrets from outbound text

PatternReplacement
API keys (sk-*, ghp_*, etc.)[REDACTED_API_KEY]
Personal emails (gmail, yahoo)[REDACTED_EMAIL]
Phone numbers[REDACTED_PHONE]
SSN patterns[REDACTED_SSN]
Credit card patterns[REDACTED_CC]
Layer 5 — Runtime Governance

"Bugs burn more money than attacks"

Cron overlaps, retry storms, and cursor bugs cost more than malicious attacks. The governor caps damage from all of them.

MechanismDefaultWhat It Catches
Spend limit$15/5min hard capExpensive loops, batch storms
Volume limit200 calls/10minRapid-fire retry bugs
Lifetime counter500 per processInfinite loops, any cause
Duplicate detection5min TTL cacheRestarts, scheduling overlaps
Layer 6 — Access Control

Path guards + URL safety

bash
# Blocked:
bash tools/security/access-control.sh check-path "~/.ssh/id_rsa"
# {"allowed": false, "reason": "Matches deny filename: .ssh"}

bash tools/security/access-control.sh check-url "http://192.168.1.1"
# {"allowed": false, "reason": "Private IP (192.168.x)"}

# Allowed:
bash tools/security/access-control.sh check-path "/workspace/memory/projects.md"
# {"allowed": true}

bash tools/security/access-control.sh check-url "https://api.openai.com"
# {"allowed": true}
Master Gate

Single entry point chaining Layer 1 + Layer 2

bash
# Full gate (sanitize + LLM scan):
echo "untrusted" | bash tools/security/gate.sh --source web

# Sanitize only (skip scanner, faster):
echo "untrusted" | bash tools/security/gate.sh --sanitize-only

# For email (fail-closed on scanner error):
echo "email" | bash tools/security/gate.sh --source email
Benefits

Before vs After

BeforeAfter
Untrusted X articles go straight to contextSanitized first, role markers stripped
Outbound messages could leak credentialsGate blocks API keys, paths, PII before send
No spend protection$15/5min hard cap + volume + lifetime limits

One failure = full compromise6 independent layers, no single point of failure
Key principle: Each layer is independent. Sanitizer catches patterns. Scanner catches semantics. Governor caps damage if both fail. Outbound gate catches leaks in the other direction. No layer depends on another working correctly.

Stop re-discovering knowledge from scratch. The agent builds and maintains a compounding wiki from every source you feed it — entity pages, concept pages, cross-references, contradiction detection.

Inspired by Andrej Karpathy's "llm-wiki" pattern (2026). "Humans abandon wikis because the maintenance burden grows faster than the value. LLMs don't get bored and can touch 15 files in one pass."

Operations

Ingest, query, lint

bash
# Ingest a new source into the wiki:
bash tools/wiki/ingest-to-wiki.sh memory/ingested/2026-04-05-article.md

# Check for contradictions across wiki:

bash tools/wiki/lint.sh

Ingest flow: A single source can touch 10-15 wiki pages. The agent reads, extracts entities, updates or creates pages, updates the index, appends to the log, and runs contradiction check. Query flow: Good answers get filed back as wiki pages — explorations compound.
